# 本体论核心概念

> 本体论不是哲学课，而是**企业知识的工程化方法**。

## 什么是本体论（Ontology）

在计算机科学中，本体论是对**某个领域中概念、属性、关系和约束的形式化描述**。

简单说：本体论回答这些问题——
- 这个领域里有哪些"东西"（概念/类）
- 这些东西有什么属性（属性/数据属性）
- 它们之间有什么关系（对象属性/关联）
- 有什么规则必须遵守（公理/约束）

## 一个直观的例子

**货代行业本体（片段）：**

```
概念层：
  Ship（船舶）
    ├── ContainerShip（集装箱船）
    └── BulkCarrier（散货船）
  Port（港口）
  Route（航线）
  Booking（订舱）
  Customer（客户）

关系层：
  Route connectsPort Port（航线连接港口）
  Ship operatesOn Route（船舶运营在航线上）
  Booking placedBy Customer（订舱由客户下达）
  Booking assignedTo Ship（订舱分配到船舶）

约束层：
  每个 Booking 必须有且仅有一个 Customer
  ContainerShip 的 TEU 容量 > 0
  Route 至少连接 2 个不同的 Port
  如果 Booking.value > 100K USD，则需要 CreditCheck = true
```

这些定义看起来简单，但它们是**机器可执行**的——可以用 OWL/RDF 编码，让 LLM 和 Agent 在运行时查询和遵守。

## 本体论 vs 其他知识组织方式

| 方式 | 结构化程度 | 推理能力 | 适用场景 |
|------|-----------|---------|---------|
| 文档/Wiki | 低 | 无 | 人类阅读 |
| 数据库 Schema | 中 | 有限（SQL） | 结构化数据存储 |
| 知识图谱 | 中-高 | 图查询 | 实体关系网络 |
| **本体论** | **最高** | **完备推理** | **领域知识的形式化规范** |
| LLM 内部知识 | 不透明 | 概率性 | 通用理解和生成 |

关键区别：**本体论定义"应该是什么"，知识图谱存储"实际是什么"，LLM 理解"语义上是什么"**。三者互补。

## 本体论的四个层次

```
Level 4: 顶层本体（Top-Level Ontology）
         — 最抽象的概念：实体、事件、过程、属性
         — 例：BFO（Basic Formal Ontology）、DOLCE、UFO

Level 3: 领域本体（Domain Ontology）
         — 特定行业的概念体系
         — 例：供应链本体、医疗本体(SNOMED)、金融本体(FIBO)

Level 2: 任务本体（Task Ontology）
         — 特定任务的知识结构
         — 例：诊断任务本体、风控任务本体、调度任务本体

Level 1: 应用本体（Application Ontology）
         — 具体系统的实例化
         — 例：OpenClaw 的工具注册表就是一个应用本体
```

## 为什么 LLM 时代更需要本体论

### 悖论：LLM 越强大，本体论越重要

- LLM 弱的时候 → 人们不用 AI 做重要决策 → 不需要治理
- LLM 强的时候 → 人们依赖 AI 做关键决策 → **必须**有治理框架
- 本体论是唯一同时满足"机器可执行"和"人类可审计"的治理基础

### LLM 不能替代本体论的三个原因

1. **一致性**：LLM 每次回答可能不同，本体论的定义是确定的
2. **可溯源**：LLM 的推理是黑盒，本体论的每条规则可以追溯到业务需求
3. **可组合**：不同的本体可以对齐和合并，LLM 的知识无法模块化管理

## 关键术语表

| 术语 | 含义 | 类比 |
|------|------|------|
| Class（类） | 一类事物的抽象 | 数据库的表 |
| Instance（实例） | 类的具体个体 | 数据库的行 |
| Property（属性） | 实例的特征 | 数据库的列 |
| Relation（关系） | 实例之间的联系 | 数据库的外键 |
| Axiom（公理） | 必须满足的规则 | 数据库的约束 |
| Ontology（本体） | 以上所有的集合 | 数据库的 Schema + 约束 + 文档 |
| OWL | 本体描述语言标准 | SQL |
| RDF | 知识图谱数据格式 | CSV/JSON |
| SPARQL | 知识图谱查询语言 | SQL SELECT |
| Protégé | 本体编辑工具 | phpMyAdmin |
